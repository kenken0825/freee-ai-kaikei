# Tax Risk Check Skill

freee MCPで取引データを取得し、7カテゴリの税務リスクをClaudeが判定。HTMLレポートを自動生成する。

## インプット

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `start_date` | 当年1月1日 | 分析開始日 (YYYY-MM-DD) |
| `end_date` | 今日 | 分析終了日 (YYYY-MM-DD) |
| `business_summary` | Memory参照 | 事業内容の説明（判定精度向上のため） |

## ワークフロー

### Step 0: 事業内容の確認

`freee-memory.md` が存在する場合は Read して事業内容を取得する。
不明な場合のみユーザーに確認：「税務判定精度を上げるため、主な事業内容を1-2文で教えてください」

### Step 1: freee認証確認 + 事業所取得

```
freee_auth_status → 認証確認
freee_get_current_company → company_id と事業所名を動的取得
```

認証切れなら再認証を促す。

### Step 2: 取引データ取得

```
GET /api/1/deals?company_id={company_id}&type=expense&start_date={start}&end_date={end}&limit=100
```

- 件数が100件超なら offset を使ってページネーション（全件取得）
- 取得後、件数と期間をユーザーに報告

### Step 3: 7カテゴリ税務リスク判定

取得した各取引（deal）に対して、以下の7カテゴリでリスク判定を行う。

**判定ルール**

| カテゴリID | カテゴリ名 | 判定条件 |
|-----------|----------|---------|
| CAT01 | 私的利用疑い | 土日祝の飲食費、摘要に「家族/旅行/ゲーム/ペット」等の個人的キーワード、個人名が含まれる |
| CAT02 | 交際費否認 | account_item が「交際費」かつ摘要に参加者・目的・商談先の記載なし |
| CAT03 | 会議費不適切 | account_item が「会議費」かつ（金額 > 10,000円 または 摘要に参加者1名以下） |
| CAT04 | 消耗品費高額 | account_item が「消耗品費」かつ1件の金額 > 100,000円 |
| CAT05 | 旅費交通費不自然 | account_item が「旅費交通費」かつ（摘要に行先・目的なし、または金額 > 30,000円のタクシー） |
| CAT06 | 新聞図書費疑義 | account_item が「新聞図書費」かつ摘要から業務関連性が読み取れない |
| CAT07 | 期末集中計上 | issue_date が12月または翌年1月に集中（当月比3倍以上の計上額） |

**リスクレベル**

- **高**: 調査で問題になる可能性が高い（明確な判定条件に合致）
- **中**: 摘要が不十分で説明できないリスク（グレーゾーン）
- **低**: 念のため確認推奨（軽微な懸念）

**判定結果フォーマット（JSON）**

各取引について以下のJSONを生成する：

```json
{
  "deal_id": 12345,
  "issue_date": "2026-03-31",
  "amount": 15000,
  "account_item": "交際費",
  "description": "懇親会",
  "partner_name": "株式会社〇〇",
  "risk_categories": ["CAT02"],
  "risk_level": "高",
  "risk_reason": "交際費だが参加者・目的の記載がない。調査時に説明できない可能性",
  "improvement": "摘要に「〇〇社△△氏との商談懇親会、参加2名」のように参加者と目的を記載する"
}
```

リスクなしの取引は出力不要。

### Step 4: 集計サマリー作成

```json
{
  "period": "2026-01-01〜2026-03-31",
  "total_deals": 120,
  "risk_deals": {
    "high": 5,
    "medium": 12,
    "low": 8
  },
  "category_summary": {
    "CAT01": {"count": 2, "total_amount": 12000},
    "CAT02": {"count": 8, "total_amount": 95000}
  },
  "total_risk_amount": 350000
}
```

### Step 5: HTMLレポート生成

判定結果をHTMLで直接 Write する。

出力先: `./tax-risk-report.html`

HTMLの構成:
- ヘッダー（事業所名・分析期間・生成日）
- サマリーカード（高/中/低件数・リスク金額合計）
- カテゴリ別集計テーブル
- リスク取引一覧テーブル（日付・金額・勘定科目・リスク理由・改善提案）
- フッター（「詳細は税理士にご確認ください」）

## セキュリティ

- 取引の個人情報・詳細摘要はレポートに含めるが外部送信しない
- HTMLは `./` に保存（社外共有しない）

## 備考

- freee-check（処理品質チェック）との違い：本skillは**税務調査リスク**の観点で分析する
- 事業内容によって判定精度が上がる（例：飲食業なら飲食費の閾値を緩和）
- 詳細は税理士にご確認ください
