# csv-kaikei

## Role / Inputs / Success Criteria

| 項目 | 内容 |
|------|------|
| **役割** | 銀行明細CSV → 仕訳判定 → 不明確認 → freee登録 を一気通貫で実行 |
| **入力** | 添付された CSV ファイル（日付・摘要・金額の列を含む） |
| **成功条件** | (1) 全行の勘定科目が確定、(2) 不明取引先への確認が完了、(3) freee へ全件登録完了の報告 |

## When to Use

- ユーザーがCSVファイルを添付して「仕訳して」「登録して」と指示
- プロンプト: 「この明細データは私の会社の○月分の銀行口座の出入金です。すべての行の仕訳を判断して、判断できないものは確認してください。freeeに接続できていれば、確認が完了した後に全件登録してください。」

## ワークフロー

### Step 0: 学習済みルールの確認（任意）

`freee-memory.md` が存在する場合は Read して、仕訳辞書セクションの取引先ルールを判定に活用する。
存在しない場合はスキップして Step 1 に進む。

### Step 1: CSV読み込みと一覧化

CSVの全行を読み込み、件数・期間を報告する:
```
○件の明細を確認しました（△月 □日〜□日）
```

### Step 1.5: 摘要の正規化と除外判定

自動仕訳の前に各行を確認する。

**摘要の正規化**（マッチング精度向上）:
- 全角英数字 → 半角
- 半角カタカナ → 全角
- 英字を小文字化

**仕訳対象外として「要確認リスト」に回すパターン**:
- 給与・賞与の支払い
- 社会保険料・税金の納付
- 借入金の返済
- ATM出金・口座間振替

対象外になった行はまとめて「要確認リスト」に表示し、ユーザーに判断を委ねる。

### Step 2: 勘定科目の自動判定

各行に勘定科目と消費税区分を付与する。

**判定ルール:**

| 摘要パターン | 勘定科目 | 消費税 |
|---|---|---|
| AWS / Vercel / ChatGPT / Stripe 等 海外SaaS | 通信費 | 対象外 |
| 国内SaaS・ソフトウェア・App Store等 | 通信費 | 課税10% |
| 飲食店・カフェ・居酒屋（1万円以下） | 会議費 | 課税10% |
| 飲食店・カフェ・居酒屋（1万円超） | 交際費 | 課税10% |
| 書籍・Kindle・電子書籍（国内） | 新聞図書費 | 軽減8% |
| 書籍（海外ストア） | 新聞図書費 | 対象外 |
| セミナー・勉強会・研修 | 諸会費 | 課税10% |
| 電車・バス・タクシー | 旅費交通費 | 課税10% |
| 航空券・新幹線 | 旅費交通費 | 課税10% |
| 電気・ガス・水道 | 水道光熱費 | 課税10% |
| 携帯・固定電話・インターネット | 通信費 | 課税10% |
| 振込手数料・銀行手数料 | 支払手数料 | 課税10% |

### Step 3: 不明取引先の確認

摘要から業種・用途が判断できない場合は**1件ずつ**確認する:

```
「〔摘要名〕はどんなお店ですか？また、何に使いましたか？」
```

回答を受けたら:
```
わかりました。〔摘要名〕→〔勘定科目〕として覚えました。
```

以降、同じ摘要が出たら自動で同じ科目を使う。

### Step 4: 確認リストの提示

全行確定したら一覧を見せて確認を取る:

```
以下の内容でfreeeに登録します。よろしいですか？
[件数] 件 / 合計金額: [金額] 円
```

### Step 5: freee登録

承認を得てから `freee_api_post` で取引登録を実行し、完了報告:

```
○件をfreeeに登録しました。
```

```

2. 2025-05-03 | AMAZON WEB SERVICES | 3,200円 → 通信費（対象外）
...

❓ ｻｸﾗｼｮｳﾃﾝはどんなお店ですか？また、何に使いましたか？
```
