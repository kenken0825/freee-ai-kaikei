# freee-memory

## Role / Inputs / Success Criteria

| 項目 | 内容 |
|------|------|
| **役割** | 会社情報・仕訳ルール・取引先を Claude Memory に保存し、来月以降の経理を省力化 |
| **入力** | プロンプト文（③今日の記憶化 / ①インタビュー形式 / ②3行入力）または自由入力 |
| **成功条件** | Memory に保存完了 + 「次回から説明不要」であることをユーザーに伝える |

## When to Use

- プロンプト③: 「今日『ｻｸﾗｼｮｳﾃﾝ』を〇〇と教えました。この判断を仕訳ルールとして覚えておいてください」
- プロンプト①: 「freee × AI 会計を始めます。私の会社の情報を覚えてほしい」
- プロンプト②: 「私の会社を覚えてください。業種：〇〇 会社名：△△…」
- または「覚えておいて」「Memoryに保存して」と直接言われたとき

## 3種類の保存モード

### モード③ — 今日の体験をそのまま記憶化（1分）

仕訳ルールを1件覚えさせる最小ステップ。

```
受け取った内容:
・取引先: ｻｸﾗｼｮｳﾃﾝ
・ユーザーの判断: 「打ち合わせのお菓子屋 → 会議費」

Memoryに保存する内容:
「ｻｸﾗｼｮｳﾃﾝ = 打ち合わせのお菓子屋 → 会議費（課税10%）として記録。
 次回同じ取引先が出たとき確認不要で自動判定する。」
```

保存後のメッセージ:
```
覚えました。次回「ｻｸﾗｼｮｳﾃﾝ」が出たとき、確認なしで会議費に分類します。
止まられるたびに「覚えておいて」と伝えると、仕訳精度が上がり続けます。
```

### モード① — 全部お任せ（チャレンジ・15分）

1問ずつインタビューして会社プロフィールを構築する。

聞く項目（順番に1問ずつ）:
1. 業種・事業内容（例：「東京でIT機器を販売しています」）
2. 会社名・屋号
3. 経理でいつも悩むこと・面倒なこと
4. よく使う取引先 3〜5社（名前と用途）
5. 毎月かならず発生する支出パターン

全項目が集まったら:
```
以下の内容で仕訳辞書の初期設定を作成してよいですか？

[整理された情報のまとめ]
```

承認後、Memoryに保存する。

### モード② — 3行だけ教える（帰宅後・5分）

ユーザーが入力したテキストをそのままMemoryに保存する。

```
受け取った情報:
業種：〇〇
会社名・屋号：△△
よく出る支出：□□
よく使う取引先：◇◇

Memoryに保存しました。次回から毎回説明しなくて大丈夫です。
```

## Memory保存方法

収集した情報を `freee-memory.md` ファイルに書き込む。

```markdown
# 私の会社のfreee仕訳メモ（AI経理アシスタント用）

## 会社プロフィール
- 業種: [業種]
- 会社名: [会社名]
- 規模: [社員数・売上規模など]

## 仕訳辞書（覚えた取引先）
| 摘要名 | 使い方 | 勘定科目 | 消費税 |
|--------|--------|----------|--------|
| [摘要名] | [使い方] | [科目] | [区分] |

## よく出る支出パターン
- [支出内容]

## メモ
[その他の業務メモ]
```

保存先: `./freee-memory.md`（このファイルは csv-kaikei や keihi-checklist スキルでも参照される）

保存後のメッセージ:
```
`freee-memory.md` に保存しました。
次回から毎回説明しなくて大丈夫です。
このファイルをAIに添付すると、すぐに仕訳を再開できます。
```
